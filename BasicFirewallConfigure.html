Basic Firewall
Configuration

Mengkonfigurasi firewall pada Linux menggunakan iptables untuk mengawal akses rangkaian dan memastikan keselamatan sistem.

01 Policy Setup

Tetapkan Policy Default DROP

Set semua chain utama INPUT, FORWARD dan OUTPUT kepada DROP untuk keselamatan maksimum. Ini bermakna semua trafik ditolak secara default kecuali yang dibenarkan secara eksplisit.

bash 
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT DROP
View
02 Loopback

Benarkan Loopback Interface

Benarkan trafik untuk loopback interface (lo) supaya sistem dapat berkomunikasi dengan dirinya sendiri — penting untuk banyak service tempatan.

bash 
iptables -A INPUT -i lo -j ACCEPT
iptables -A OUTPUT -o lo -j ACCEPT
View
03 SSH Access

Benarkan Trafik Masuk SSH (Port 22)

Benarkan sambungan SSH masuk. Ini penting untuk remote access ke server dari luar rangkaian tempatan.

bash 
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
View
04 Web Traffic

Benarkan Trafik HTTP & HTTPS

Benarkan trafik web masuk melalui port 80 (HTTP) dan port 443 (HTTPS) supaya server boleh serve laman web kepada pengguna.

bash 
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
View
05 ICMP / Ping

Benarkan Trafik ICMP (Ping)

Benarkan permintaan ping masuk. Berguna untuk diagnosa rangkaian — memastikan server boleh dihubungi dari luar.

bash 
iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT
View
06 Outbound

Benarkan Trafik Keluar

Benarkan semua trafik keluar dari server. Ini penting supaya server boleh membuat request keluar seperti update pakej dan DNS resolution.

bash 
iptables -A OUTPUT -j ACCEPT
View
07 Persist Rules

Simpan Rules iptables

Simpan semua rules supaya kekal selepas server reboot. Tanpa langkah ini, semua rules akan hilang bila sistem dimulakan semula.

bash 
iptables-save > /etc/iptables/rules.v4
View
08 Verify

Lihat Rules Semasa

Paparkan semua rules iptables yang aktif dengan maklumat terperinci termasuk packet count dan byte count untuk setiap rule.

bash 
iptables -L -v -n --line-numbers
View

Summary.

Firewall yang dikonfigurasi ini menerapkan prinsip least privilege — semua trafik ditolak secara default, dan hanya port yang diperlukan (22, 80, 443) dibuka secara eksplisit.
